Names through the social graph

GNS is a decentralized naming and discovery layer built on the Nostr social graph. Instead of mapping names to servers or directly to keys, it resolves names through graph relationships.

Try the demo ↓ View on GitHub
DNS
name → server

Globally unique. Pay to own.

NIP-05
name → pubkey

Verification via a domain.

GNS
(name, graph position) → pubkey

Discovery via relationships.

The scarce resource is your position in the graph

Names need not be globally unique — john.company, john.friend, and john.bitcoin can all coexist. A GNS address reads right-to-left as a walk from your vantage point:

barbara@alex.michael.nostr
barbara — target name alex.michael — graph path .nostr — namespace

You → follow michael → in his follows find alex → in hers find barbara. Each hop is an existing Nostr kind 3 follow; the graph is the resolver.

Resolve a name, live

Runs entirely in your browser on a built-in example graph — no server, no relays. Mirrors the real resolver's normalization, membership, and ambiguity rules.

Example graph: You follow Michael, Carol, and two accounts both named “Frank”. Michael follows Alex & Dave; Alex, Carol, Dave each follow Barbara.

    GNS is an open prototype. To run the real resolver (live relays + API + this dashboard): 

    git clone https://github.com/asherp/GNS
cd GNS
cargo run -- --demo   # then open http://127.0.0.1:8080/
